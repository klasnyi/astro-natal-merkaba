# Astro Natal МерКаБа

Cowork-скил для построения натальной астрологической карты по дате, времени и месту рождения. Поддерживает **западную** (тропический зодиак, система домов Плацидус) и **ведическую** (сидерический Лахири, 27 накшатр, Whole Sign) системы.

Расчёты — через [Swiss Ephemeris](https://www.astro.com/swisseph/) (JPL DE431, точность <0.001°). На выходе — стилизованный DOCX 15-25 страниц с колесом карты, архетипической интерпретацией планет, аспектов, домов, накшатр и практическими рекомендациями.

---

## ⚠️ Дисклеймер

Скил даёт **архетипический инструмент саморефлексии**, а не предсказание событий, медицинский диагноз или психологическую консультацию. Натальная карта описывает энергетические тенденции и точки роста — не судьбу. Все жизненные решения — за вами.

## Что делает скил

### Поддерживаемые системы

| Система | Зодиак | Дома | Особенности |
|---|---|---|---|
| **Западная** | Тропический | Плацидус | Транзитная астрология, психология |
| **Ведическая** | Сидерический Лахири (~23.79°) | Whole Sign | 27 накшатр, ведические даши |
| **Солнечная** (без времени) | По выбору | — | Только знаки планет, без ASC и домов |

### Что входит в отчёт

- 🌞 **Солнце-Луна-Асцендент** — синтез ядра карты
- 🪐 **13 планет** (Солнце → Меркурий → Венера → Марс → Юпитер → Сатурн → Уран → Нептун → Плутон → Северный Узел → Хирон → Лилит) — каждая 8-15 предложений с привязкой к градусу, дому, диспозитору, аспектам
- 🏠 **Все 12 домов** (если время известно) — управитель, планеты в доме, темы
- ⭐ **10+ ключевых аспектов** (орб ≤3°) — каждый с расшифровкой 6-10 предложений
- 🔥💧🌍💨 **Стихии и модальности** — % распределение, дефицит, компенсация
- 🌌 **Стеллиумы** — если есть, отдельный анализ
- 🕉 **Накшатры** (ведический режим) — Луна и Асцендент с символами, божествами
- 🎯 **8-12 практических рекомендаций** — конкретно под карту, не общие

### Архитектура

Двухскриптовая:

```
scripts/
├── build_chart.py    # Расчёт: kerykeion → JSON + matplotlib wheel PNG
└── render_docx.py    # DOCX из chart.json + interp.json (стиль МерКаБа)

references/           # Справочники архетипов (НЕ готовые интерпретации)
├── planets.json
├── signs.json
├── houses.json
├── aspects.json
└── nakshatras.json
```

**Интерпретации генерирует Claude** в момент составления отчёта — синтезируя `chart.json` с reference-файлами по промту в `SKILL.md`. Не pre-written: каждая карта получает уникальный текст.

## Использование

### В Cowork — chat-native flow

Скил спрашивает обычным сообщением (без модальных окон):

> Для натальной карты нужно 5 данных:
> 1. **Дата рождения** (ДД.ММ.ГГГГ)
> 2. **Время рождения** (ЧЧ:ММ) — или скажи "время неизвестно"
> 3. **Город рождения**
> 4. **Имя**
> 5. **Система** — западная / ведическая / обе

Можно ответить одним сообщением — скил распарсит.

### Локально через CLI

```bash
python3 scripts/build_chart.py \
  --date 13.08.1993 \
  --time 5:30 \
  --city "Москва" \
  --name "Дмитрий" \
  --system western \
  --outdir /tmp/astro-dima
```

Параметры:
- `--date` — `ДД.ММ.ГГГГ`
- `--time` — `ЧЧ:ММ` (опционально; если нет — солнечная карта)
- `--city` — название города на любом языке (геокодируется через Nominatim)
- `--name` — имя для шапки отчёта
- `--system` — `western` / `vedic`
- `--outdir` — куда сложить результаты

Затем рендер DOCX:

```bash
# Сначала вы (или Claude) пишете interp.json с интерпретациями по промту в SKILL.md
python3 scripts/render_docx.py \
  --chart /tmp/astro-dima/Дмитрий_1993-08-13.json \
  --interp /tmp/astro-dima/interp.json \
  --out /tmp/astro-dima/Дмитрий_1993-08-13_natal.docx
```

## Эталонная карта (smoke test)

Тестовые данные: **13.08.1993, 05:30, Москва, западная система**.

Ожидаемое ядро карты:

| Параметр | Значение |
|---|---|
| ☉ Солнце | Лев 20.32°, дом 1 |
| ☽ Луна | Близнецы 18.34°, дом 11 |
| ↑ Асцендент | Лев 14.68° |
| ⊕ MC | Овен 23.35° |
| Доминанта стихий | Воздух 4 · Огонь 3 · Земля 2 · Вода 2 |

**Сверхточные аспекты (орб <1°):**
1. Уран ☌ Нептун — 0.07° (Козерог, 6 дом — поколенческая тема)
2. Сатурн ☍ Хирон — 0.38° (Водолей 7 ↔ Лев 1 — центральная ось)
3. Меркурий △ Лилит — 0.9° (Лев 12 / Овен 9)

Если ваш прогон даёт другие значения — что-то сломалось (см. секцию Точность ниже).

## Качество интерпретаций — стандарты

Закреплены в `SKILL.md`:

- **8-15 предложений на планету** с привязкой к градусу, дому, диспозитору, аспектам
- **10+ аспектов с орбом ≤3°**, каждый 6-10 предложений с движением (applying/separating) и интенсивностью
- **Все 12 домов** (если `has_time=true`)
- **Стеллиумы** — если есть, отдельный анализ
- **Стихии и модальности** — % с расшифровкой дефицита и компенсаторных практик
- **Итог** — DOCX **15-25 страниц**, ~15-25 тыс. слов

Конкретика обязательна:
- ❌ «У вас сильное Солнце»
- ✅ «Солнце во Льве 20.32° в 1.5° от ASC, в соединении с Хироном 27° — ядро идентичности слито с маской и раной»

## Точность

| Параметр | Значение |
|---|---|
| Эфемерида | Swiss Ephemeris (JPL DE431) |
| Точность планет | <0.001° |
| Геокодинг | Nominatim (OpenStreetMap) |
| Таймзоны | timezonefinder (оффлайн БД с историческими TZ) |
| ASC/MC формула | Истинное местное звёздное время |
| Дома (западная) | Алгоритм Плацидуса (Meeus, Astronomical Algorithms) |
| Аянамша (ведическая) | Лахири (Chitrapaksha) |

Чтобы проверить точность — сравните с [astro.com](https://www.astro.com/horoscope) (бесплатный профессиональный калькулятор).
